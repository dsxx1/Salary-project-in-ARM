### 1.2. Соглашения о нотации

Все числовые диапазоны в документе используют **полуоткрытый интервал [нижняя граница; верхняя граница)**, то есть нижняя граница **включается**, верхняя **не включается**.

Примеры:
- «88–93%» означает **[88%; 93%)** — ровно 88% входит, ровно 93% — нет
- «93–98%» означает **[93%; 98%)** — ровно 93% входит в этот диапазон

**Округление:** ❓ УТОЧНИТЬ — до копеек или до рубля? Как округляет финансист вручную сейчас? (от ответа зависит тест «расхождение 0%»)

## 2. ТРЕБОВАНИЯ К ВЫПОЛНЕНИЮ РАБОТ

### 2.1. Общая информация — устраняемые проблемы

| № | Проблема | Решение в Системе |
|---|---|---|
| №1 | Смена привязана к юрлицу | Привязка смены к должности |
| №2 | Нет табеля у автоломбарда | Единый табель для всех ОП |
| №3 | Нет контроля замен | Документ закрепления за точкой |
| №4 | Можно пересчитать закрытый период | Статусы периода + запрет изменений |
| №5 | Нет плана → нет расчёта | Правило «нет плана = 100%» при наличии факта |
| №6 | Баллы теряются при смене юрлица | Баллы привязаны к связке «сотрудник + должность» |
| №7 | Нет истории изменений | Журнал действий + версионирование расчётов |
| №8 | Оклад не привязан к точке | Формула: Ставка_ОП × часы × Крт |

### 2.2. Описание Работ

#### 2.2.1. Архитектура Системы

Система работает по следующему пайплайну:

1. **Получение данных из АРМ.** Из АРМ поступают сырые данные: табель (смены), планы ТО, баллы сотрудников.
2. **Конструктор витрины.** Серверный модуль агрегирует сырые данные в единую структуру без необходимости написания SQL-запросов пользователем.
3. **Rule Engine (Ядро расчёта).** Программный модуль, реализующий всю математику Приказа СКС-03. Все параметры (ставки, коэффициенты, шкалы) хранятся в базе данных и редактируются через экран «Настройки» без изменения кода.
4. **Интерфейс.** Веб-интерфейс для запуска расчёта, просмотра ведомостей, личного кабинета сотрудника и отчётов.

#### 2.2.2. Экран 1 — Табель

##### 2.2.2.1. Логика открытия смены

Сотрудник открывает смену в АРМ:
- Выбирает должность из своего списка доступных должностей.
- Подтверждает ОП (точку).
- Смена фиксируется в Системе.

##### 2.2.2.2. Состав данных смены

Каждая смена содержит:
- Сотрудник, должность, дата.
- ОП (точка), ТО (группа точек).
- Юрлицо (только для документов — не влияет на расчёт ЗП).
- Часов отработано.
- Тип смены: **обычная** / **замена**.

##### 2.2.2.3. Ограничения для мобильного товароведа

Для должности «Мобильный товаровед» действуют лимиты:
- Максимум **3 смены** на одном ОП за месяц.
- Максимум **17 смен** за месяц.

При превышении лимита Система блокирует открытие смены и отображает остаток доступных смен.

> Примечание: должность «Мобильный товаровед» на момент написания ТЗ отсутствует в справочнике. Функционал реализуется заранее.

#### 2.2.3. Экран 2 — Закрепление за точкой

##### 2.2.3.1. Назначение закреплений

Каждый сотрудник закрепляется за основной точкой. Супервайзер создаёт документ закрепления и самостоятельно закрепляет своих сотрудников.

##### 2.2.3.2. Логика проверки при открытии смены

| Ситуация | Действие Системы |
|---|---|
| Точка входит в список закрепления | Смена открывается как **обычная** |
| Точка не входит в список | Запрос «Это замена?» |
| Ответ «Да» | Смена открывается с флагом **замена** → применяется доплата +42 руб/ч |
| Ответ «Нет» | Смена не открывается |

#### 2.2.4. Экран 3 — Планы ТО

Финансист вводит планы ТО по следующим показателям в разрезе ТО и месяца:

| Показатель | Описание | Связь с расчётом ЗП |
|---|---|---|
| **Актив** | Объём актива, тыс. руб. | ❓ УТОЧНИТЬ: Па или Пв? |
| **Вал** | Валовая выручка, тыс. руб. | ❓ УТОЧНИТЬ: Па или Пв? |
| **Скупка** | Объём скупки золота, гр | → ПсЗ (премия за скупку золота) |
| **Техника** | Объём актива техники, тыс. руб. | → Шаг 4, надбавка за актив техники |

При отсутствии плана по показателю применяется правило «нет плана = 100%» (см. п. 2.2.6.1).

##### 2.2.5.1. Начисление начальных баллов

#### 2.2.6. Экран 5 — Расчёт ЗП

##### 2.2.6.1. Правило «Нет плана = 100%»

| Ситуация | Действие Системы |
|---|---|
| Есть факт, нет плана | Выполнение = **100%** автоматически. В расчётном листе: «план не задан, применено 100%» |
| Есть факт, план = 0 | Выполнение = **0%** (намеренное решение, не ошибка) |
| Нет факта, нет плана | **Ошибка**, расчёт заблокирован. Сообщение: «По ТО [название] нет фактических данных» |

##### 2.2.6.2. Шаг 1 — Оклад

```
Оклад = Σ (Ставка_ОП × часы_смены × Крт) + Нвл × число_смен
```

`Ставка_ОП` берётся из справочника «Ставки_ОП» по связке: **точка + должность**.

**Пример расчёта:**
```
80 ч на ОП «Спортивная» (150 руб/ч) +
40 ч на ОП «Центральный» (170 руб/ч), Крт = 1,1,
Нвл = 150 руб/смена × 20 смен
= (80 × 150 × 1,1) + (40 × 170 × 1,1) + 3 000 = 23 680 руб.
```

> ❓ УТОЧНИТЬ: границы 90%/91% и 100%/101% — в исходном ТЗ «70–90% → 2%, 91–100% → 2,5%». Что при ровно 90% и ровно 100%? Здесь принято [70%; 91%) = 2%, [91%; 101%) = 2,5%. Подтвердить.

##### 2.2.6.6. Шаг 5 — МГВ (выполняется последним)

```
Итоговая ЗП = MAX(результат Шагов 1–4, МГВ)
```

**Условие применения:** сотрудник отработал ≥ 14 **календарных** дней.

Если сотрудник отработал **< 14 календарных дней** — МГВ не применяется, берётся результат Шагов 1–4.

| Тип МГВ | Ставка | Максимум |
|---|---|---|
| Обычный | 310 руб/ч × часы | 55 800 руб/мес |
| Мобильный товаровед | 390 руб/ч × часы | 70 200 руб/мес |

Нвл и Крт к МГВ **не применяются**.

##### 2.2.6.7. Особые должности

**Мобильный товаровед:**
```
Посн = 10% от премии ОП за смену + ПсЗ
```
Кн не применяется.

**Подменный товаровед (декретница):**
- ≥ 8 смен → оклад по стандартной ставке ОП
- < 8 смен → **175 руб/ч** фиксированно

**Менеджер автоломбарда (автономный):**
```
ЗП = Оклад × КЧ × Крт + (Па + Пв + Пвыд) × Кн / КРЧ × КЧ
```
Кн для автоломбарда:

| Сумма займов, тыс. руб. | Кн |
|---|---|
| [0; 700) | 1,0 |
| [700; 1 200) | 1,5 |
| [1 200; ∞) | 2,0 |

**Менеджер автоломбарда (неавтономный):**
```
ЗП = Оклад × КЧ × Крт  (без премии)
```

#### 2.2.7. Экран 6 — Отчёты

В разделе отчётов должны быть доступны:

- Таблицы и графики.
- Сводные таблицы (настраиваются через фильтры — без drag-and-drop полей).
- Экспорт в Excel.
- Отдельный отчёт по заменам: кто, на какой ОП, когда выходил.

#### 2.2.8. Экран 7 — Личный кабинет сотрудника

Сотрудник видит только свои данные (только просмотр):
- Расшифровка ЗП: оклад, доплаты, премии, МГВ.
- Баллы и категория (Крт).
- Табель за выбранный период.
- Причина применения МГВ (если применялась).

#### 2.2.9. Экран 8 — Настройки (Конструктор справочников)

ФЭО редактирует через интерфейс без кода:

| Справочник | Что редактируется |
|---|---|
| Ставки_ОП | Ставки по связке: точка + должность |
| Таблицы премий | Па+Пв по % выполнения плана |
| Проценты продаж | КМТ, КМЗ, скупка техники |
| Коэффициенты Кн | Таблица весов металла |
| Шкала Нвл | Надбавка по стажу |
| Правила баллов | Начисление/списание по показателям |

Все изменения справочников фиксируются в **истории изменений** с указанием автора, даты и версии.

#### 2.2.10. Экран 9 — Журнал действий

Администратор видит:
- Кто, когда, какое действие совершил в Системе.
- Версии расчётов (v1, v2, …) с возможностью сравнения.
- Историю разблокировок периодов с указанием причины.

#### 2.2.11. Права доступа

| Роль | Права |
|---|---|
| **Сотрудник** | Личный кабинет — только свои данные, только просмотр |
| **Супервайзер** | Смены своих сотрудников, закрепления, ходатайства, отчёты |
| **Финансист** | Расчёт ЗП, планы, баллы, ведомости |
| **ФЭО** | Всё, что Финансист + настройка справочников + разблокировка периодов + управление правами |
| **Администратор** | Полный доступ |
