## 2. ТРЕБОВАНИЯ К ВЫПОЛНЕНИЮ РАБОТ

### 2.1. Общая информация

Существующий процесс расчёта заработной платы характеризуется следующими проблемами, которые Система должна устранить:

| № проблемы | Описание проблемы | Решение в Системе |
|---|---|---|
| №1 | Смена привязана к юрлицу | Привязка смены к должности |
| №2 | Нет табеля у автоломбарда | Единый табель для всех ОП |
| №3 | Нет контроля замен | Документ закрепления за точкой |
| №4 | Можно пересчитать закрытый период | Статусы периода + запрет изменений |
| №5 | Нет плана → нет расчёта | Правило нет плана = 100% если есть продажи |
| №6 | Баллы теряются при смене юрлица | Баллы привязаны к связке сотрудник + должность |
| №7 | Нет истории изменений | Журнал действий + версионирование расчётов |
| №8 | Оклад не привязан к точке | Формула: Ставка_ОП × часы × Крт |

Реализация указанного функционала направлена на обеспечение прозрачности, контроля и автоматизации расчёта заработной платы, минимизацию ручного труда и исключение ошибок, связанных с человеческим фактором.

### 2.2. Описание Работ

#### 2.2.1. Архитектура Системы

Система должна работать по следующему пайплайну:

1. **Получение данных из АРМ.** Из АРМ поступают сырые данные: табель (смены), планы ТО, баллы сотрудников.
2. **Конструктор витрины.** Серверный модуль агрегирует сырые данные в единую структуру (витрину данных) без необходимости написания SQL-запросов пользователем.
3. **Rule Engine (Ядро расчёта).** Программный модуль, реализующий всю математику Приказа СКС-03, выполняет расчёт заработной платы.
4. **Интерфейс.** Веб-интерфейс для запуска расчёта, просмотра ведомостей, личного кабинета сотрудника и отчётов.

#### 2.2.2. Экран 1 — Табель

##### 2.2.2.1. Новая логика открытия смены

Сотрудник открывает смену в АРМ, при этом:

- Выбирает должность из своего списка доступных должностей.
- Подтверждает ОП (точку).
- Смена фиксируется в Системе.

##### 2.2.2.2. Состав данных смены

Каждая смена должна содержать:

- Сотрудник, должность, дата.
- ОП (точка), ТО (группа точек).
- Юрлицо (только для документов — не влияет на расчёт).
- Часов отработано.
- Тип смены: обычная / замена.

##### 2.2.2.3. Ограничения для мобильного товароведа (согласно мотивации, пока нет такой должности)

Для должности «Мобильный товаровед» должны действовать лимиты:

- Максимум 3 смены на одном ОП за месяц.
- Максимум 17 смен за месяц.

При превышении лимита Система блокирует открытие смены и отображает остаток доступных смен.

#### 2.2.3. Экран 2 — Закрепление за точкой

##### 2.2.3.1. Назначение закреплений

Каждый сотрудник закрепляется за основной точкой. Супервайзер создает документ закрепления за точкой и закрепляет своих сотрудников сам.

##### 2.2.3.2. Логика проверки при открытии смены

При открытии смены Система проверяет точку:

- **Точка из списка закрепления** → смена открывается как обычная.
- **Точка не из списка закрепления** → Система выводит запрос «Это замена?».
  - Ответ «Да» → смена открывается с флагом «замена», применяется доплата +42 руб/ч.
  - Ответ «Нет» → смена не открывается.

#### 2.2.4. Экран 3 — Планы ТО

Финансист должен иметь возможность вводить планы ТО по следующим показателям:

- Актив.
- Вал (валовая выручка).
- Скупка.
- Техника.

Планы вводятся в разрезе ТО и месяца. В случае отсутствия плана по какому-либо показателю действует правило «нет плана = 100%» (см. п. 2.2.6.1).

#### 2.2.5. Экран 4 — Баллы и коэффициент рейтинга

##### 2.2.5.1. Начисление баллов

При трудоустройстве сотруднику автоматически начисляется +25 баллов, ввод начальных баллов исходит из документа назначенияя в должность.

Максимальные значения баллов:

- Товаровед-кассир — 70 баллов.
- Продавец КМТ — 50 баллов.

##### 2.2.5.4. Правила работы с баллами

Баллы привязаны к связке «Сотрудник + Должность». При смене юрлица баллы не теряются. Изменение баллов за закрытый период запрещено.

#### 2.2.6. Экран 5 — Расчёт ЗП

##### 2.2.6.1. Правило «Нет плана = 100%»

- **Есть факт, нет плана** → выполнение = 100% автоматически. В расчётном листе отображается пометка: «план не задан, применено 100%».
- **Есть факт, план = 0** → выполнение = 0% (намеренное решение).
- **Нет факта, нет плана** → ошибка, расчёт заблокирован. Сообщение: «По ТО [название] нет фактических данных».

##### 2.2.6.2. Шаг 1 — Оклад

Оклад рассчитывается по формуле:

```
Оклад = Σ (Ставка_ОП × часы_смены × Крт) + Нвл × число_смен
```

Ставка_ОП берётся из справочника «Ставки_ОП» по связке: точка + должность.

**Пример расчёта:**

```
80 ч на ОП «Спортивная» (150 руб/ч) + 
40 ч на ОП «Центральный» (170 руб/ч), Крт = 1,1, 
Нвл = 150 руб/смена × 20 смен
= (80 × 150 × 1,1) + (40 × 170 × 1,1) + 3000 = 23 680 руб.
```

##### 2.2.6.3. Шаг 2 — Доплаты к окладу

- **Замена в другой нас. пункт:** +42 руб/ч (к сменам с флагом «замена»).
- **Сверхнорма (с 18-й смены):** +42 руб/ч (Крт не применяется).
- **Старший товаровед:** +115 руб/ч (не более 20 000 руб/мес). (возможно использовать флаг старший товаровед)

##### 2.2.6.4. Шаг 3 — Премия

Премия рассчитывается по формуле:

```
Посн = (Па + Пв) × Кн / КРЧ × КЧ + ПсЗ
```

**Па и Пв** зависят от % выполнения плана ТО:

| 93–98% | 4 000 |
| 98–103% | 10 000 |
| 103–108% | 14 000 |
| 108%+ | 16 000 |

**ПсЗ (скупка золота, руб/гр):**

##### 2.2.6.5. Шаг 4 — Дополнительные премии

- **Автоломбард:** +500 руб/анкету, +1500 руб. за первую, +3% или +4% от суммы выдачи.
- **Техника КМТ (% от личных продаж):**
  - до 80% → 5% / 81–90% → 7% / 91–120% → 10% / 121%+ → 12% (для товароведа: 3% / 5% / 10% / 12%).
- **Актив техники:**
  - 88–93% → 800 руб. / 93–98% → 1500 руб. / 98–103% → 3500 руб. / 103–108% → 5500 руб. / 108%+ → 8000 руб.
- **Скупка техники:** 5% (СБП) или 3% (наличные).
- **Золото КМЗ (% от личных продаж):**
  - <50% → 1% / 50–70% → 1,5% / 70–90% → 2% / 91–100% → 2,5% / 100–120% → 4% / 120–140% → 4,5% / 140–150% → 4,8% / 150%+ → 5%.

##### 2.2.6.6. Шаг 5 — МГВ (выполняется последней)

Условие применения: сотрудник отработал ≥14 календарных дней.

Итоговая ЗП = MAX(результат шагов 1–4, МГВ).

- **МГВ обычный:** 310 руб/ч × часы (максимум 55 800 руб.).
- **МГВ мобильного:** 390 руб/ч × часы (максимум 70 200 руб.).

Нвл и Крт к МГВ не применяются.

##### 2.2.6.7. Правила для особых должностей

**Мобильный товаровед:**
Посн = 10% от премии ОП за смену + ПсЗ. Кн не применяется.

**Подменный товаровед (декретница):**
- 8+ смен → оклад по стандартной ставке.
- Менее 8 смен → 175 руб/ч фиксированно.

**Менеджер автоломбарда (автономный):**
ЗП = Оклад × КЧ × Крт + (Па + Пв + Пвыд) × Кн / КРЧ × КЧ.
Кн: до 700 тыс. → 1 / до 1,2 млн → 1,5 / выше → 2.

**Менеджер автоломбарда (неавтономный):**
ЗП = Оклад × КЧ × Крт (без премии).

#### 2.2.7. Экран 6 — Отчёты

В разделе отчётов должны быть доступны:

#### 2.2.8. Экран 7 — Личный кабинет сотрудника

Сотрудник должен видеть:

- Свои баллы и категорию.

#### 2.2.9. Экран 8 — Настройки (Конструктор справочников)

ФЭО должен иметь возможность через интерфейс редактировать:

- Коэффициенты Кн.
- Шкалу Нвл.

Все изменения справочников должны фиксироваться в истории изменений.

#### 2.2.10. Экран 9 — Журнал действий

Администратор должен видеть:

- Кто, когда, какое действие совершил в Системе.
- Версии расчётов (v1, v2, …).
- Историю разблокировок периодов с указанием причины.

## 3. ПОРЯДОК ПРОВЕДЕНИЯ ПРИЕМОЧНЫХ ИСПЫТАНИЙ

Для проведения приемочных испытаний должна быть предоставлена следующая документация:

- Программа и методика испытаний.
- Протокол приемочных испытаний.

В программе и методике испытаний указывается описание контрольных тестов в соответствии с разделом 2 настоящего технического задания.

Проект протокола содержит:

- Предмет испытаний.
- Цель испытаний.
- Перечень пунктов программы и методики испытаний, по которым проведены испытания.

Работы по проведению испытаний не должны оказывать влияния на функционирование подсистем, не участвующих в испытаниях.

**Контрольные тесты (основные):**
